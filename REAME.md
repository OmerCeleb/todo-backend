# Todo Backend API

A modern, RESTful Todo application backend built with Spring Boot and PostgreSQL. This project demonstrates clean architecture principles with proper separation of concerns across Entity, Repository, Service, and Controller layers.

## Features

- **CRUD Operations**: Complete Create, Read, Update, Delete functionality for todos
- **Priority Management**: Support for LOW, MEDIUM, HIGH priority levels
- **Category Organization**: Organize todos by custom categories
- **Due Date Tracking**: Set and track due dates for todos
- **Search & Filter**: Advanced filtering by completion status, priority, category
- **Statistics**: Get comprehensive stats about your todos
- **Data Validation**: Robust input validation using Bean Validation
- **Auto Timestamps**: Automatic creation and update timestamps

## Technology Stack

- **Java 21** - Modern Java with latest features
- **Spring Boot 3.3.0** - Enterprise-grade framework
- **Spring Data JPA** - Data persistence layer
- **PostgreSQL** - Robust relational database
- **Hibernate** - ORM for database operations
- **Maven** - Dependency management
- **Docker** - Containerized PostgreSQL instance

## Architecture

The project follows a layered architecture pattern:

```
├── Entity Layer (Todo.java)
│   └── Data model and database mapping
├── Repository Layer (TodoRepository.java)
│   └── Database queries and data access
├── Service Layer (TodoService.java)
│   └── Business logic and data transformation
├── Controller Layer (TodoController.java)
│   └── REST API endpoints and HTTP handling
└── DTO Layer (TodoRequestDTO, TodoResponseDTO, TodoStatsDTO)
    └── Data transfer objects for API communication
```

## Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/todo-backend.git
cd todo-backend
```

### 2. Start PostgreSQL with Docker

```bash
docker run --name todo-postgres \
  -e POSTGRES_USER=omer \
  -e POSTGRES_PASSWORD=Blabla123 \
  -e POSTGRES_DB=mydb \
  -p 5432:5432 \
  -d postgres:15
```

### 3. Configure Database

The application is pre-configured to connect to PostgreSQL. Update `src/main/resources/application.yml` if needed:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/mydb
    username: omer
    password: Blabla123
```

### 4. Build and Run

```bash
# Build the project
mvn clean compile

# Run the application
mvn spring-boot:run
```

The application will start on `http://localhost:8080`

## API Documentation

### Base URL
```
http://localhost:8080/api/todos
```

### Endpoints

#### Get All Todos
```http
GET /api/todos
```

**Query Parameters:**
- `completed` (boolean): Filter by completion status
- `priority` (string): Filter by priority (LOW, MEDIUM, HIGH)
- `category` (string): Filter by category
- `search` (string): Search in title and description

#### Get Todo by ID
```http
GET /api/todos/{id}
```

#### Create New Todo
```http
POST /api/todos
Content-Type: application/json

{
  "title": "Complete project documentation",
  "description": "Write comprehensive README and API docs",
  "priority": "HIGH",
  "category": "Documentation",
  "dueDate": "2025-01-25T23:59:59"
}
```

#### Update Todo
```http
PUT /api/todos/{id}
Content-Type: application/json

{
  "title": "Updated title",
  "description": "Updated description",
  "priority": "MEDIUM",
  "category": "Updated Category"
}
```

#### Toggle Todo Completion
```http
PATCH /api/todos/{id}/toggle
```

#### Delete Todo
```http
DELETE /api/todos/{id}
```

#### Get Statistics
```http
GET /api/todos/stats
```

**Response:**
```json
{
  "total": 10,
  "completed": 4,
  "active": 6,
  "overdue": 2,
  "completionPercentage": 40.0
}
```

#### Additional Endpoints

- `GET /api/todos/categories` - Get all categories
- `GET /api/todos/overdue` - Get overdue todos
- `GET /api/todos/due-today` - Get todos due today
- `GET /api/todos/completed` - Get completed todos
- `GET /api/todos/active` - Get active todos
- `DELETE /api/todos/bulk` - Bulk delete todos

## Data Model

### Todo Entity

```java
{
  "id": 1,
  "title": "Task title",
  "description": "Task description",
  "completed": false,
  "priority": "HIGH",
  "category": "Work",
  "createdAt": "2025-01-20T10:00:00",
  "updatedAt": "2025-01-20T10:00:00",
  "dueDate": "2025-01-25T23:59:59"
}
```

### Priority Levels
- `LOW` - Low priority tasks
- `MEDIUM` - Medium priority tasks (default)
- `HIGH` - High priority tasks

## Database Schema

The application automatically creates the following table structure:

```sql
CREATE TABLE todos (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    completed BOOLEAN NOT NULL DEFAULT FALSE,
    priority VARCHAR(255) NOT NULL CHECK (priority IN ('LOW','MEDIUM','HIGH')),
    category VARCHAR(100),
    created_at TIMESTAMP(6) NOT NULL,
    updated_at TIMESTAMP(6) NOT NULL,
    due_date TIMESTAMP(6)
);
```

## Configuration

### Database Configuration

Update `application.yml` to customize database settings:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/your_db
    username: your_username
    password: your_password
  jpa:
    hibernate:
      ddl-auto: update  # Use 'create-drop' for testing
    show-sql: true      # Disable in production
```

### Server Configuration

```yaml
server:
  port: 8080  # Change if needed
```
